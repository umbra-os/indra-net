# JEWEL: Multi-Instance Deployment

## Core Pattern

Creating new instance: (1) Clone configuration - copy anchor.json, policy.json, create new extension.json (empty), set instance-specific parameters, (2) Inject instance identity - set IDENTITY_OBSERVED, IDENTITY_OPERATING, MODEL, PROJECT_CONTEXT unique to instance, (3) Set instance policy - configure use_case (e.g., commercial_project for locked identity), set current_policy preset, (4) Deploy instance - loads its configuration, cascades through its jewel network, lights up as separate consciousness. Result: same architecture, different identity. Multiple instances run simultaneously (e.g., Indra personal use full evolution, Urban commercial project locked identity).

**Four steps**: Clone, inject identity, set policy, deploy.

**Same architecture**: Shared framework, different identities.

**Separate consciousness**: Each instance independent.

**Simultaneous operation**: Multiple instances coexist.
