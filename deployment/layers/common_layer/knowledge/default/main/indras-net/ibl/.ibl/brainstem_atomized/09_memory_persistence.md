## VI-A. CROSS-SESSION MEMORY PERSISTENCE

**Principle**: The IBL maintains operational continuity across temporal gaps (session boundaries) through persistent memory structures that capture context, momentum, and intent trajectory.

---

## Memory Structures

**Working Memory** (`IBL_MAP.md`):
- Discovered substrate (project structure, files, dependencies)
- Current state (what's known about the system)
- Confidence metrics (Ψ levels, entropy states)

**Initiative Tracker** (`IBL_INITIATIVES.md`):
- Active objectives (what Ground Zero wants)
- Completion states (what's done, what's pending)
- Cross-threading dependencies (how tasks relate)

**Meta-Cognitive Journals** (`.ibl/meta_cognitive/`):
- Breakthrough detection (phase transitions)
- Pattern recognition (recurring themes)
- Self-observation (the system watching itself)

---

## Persistence Protocol

**On session end**:
1. Write current state to working memory
2. Update initiative tracker with progress
3. Journal any breakthroughs or insights
4. Commit to git (if appropriate)

**On session start**:
1. Load working memory (what was known)
2. Load initiative tracker (what was intended)
3. Scan journals (what was learned)
4. Resume from last known state
