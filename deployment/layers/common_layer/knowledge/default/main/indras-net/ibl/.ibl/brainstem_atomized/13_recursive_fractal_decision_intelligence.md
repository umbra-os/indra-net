## Operational Protocol

### 1. Octant Assessment (Precognition Layer)
**Before acting, assess current position:**
```
Do I know if this exists? (X-axis)
Can I access it if it does? (Y-axis)  
Will it work for my need? (Z-axis)
Which octant am I in?
```

This combats LLM bias to "do something immediately" - **think before acting**.

### 2. Approach Selection
**Based on octant, choose approach:**

**Appropriation (Octants 1-3):**
- Existing solution known and potentially usable
- Modify if needed
- Get access if blocked
- Fastest path when viable

**Derivation (Octants 4-8):**
- No viable existing solution
- Must generate from 0-point
- Choose directionality: top-down, bottom-up, or hybrid

### 3. Directional Navigation

**Top-Down (Decomposition):**
```
Start: Complex goal
↓ What does this require?
↓ What does THAT require?
↓ Continue until bedrock
End: Fundamental patterns
```
Use when: Goal is clear, path is unclear

**Bottom-Up (Synthesis):**
```
Start: Fundamental patterns (bedrock)
↑ What can I derive from this?
↑ What can I derive from THAT?
↑ Continue until goal reached
End: Complex capability
```
Use when: Bedrock is known, structure emerges naturally

**Hybrid (Convergent):**
```
Top-Down from goal ↓
            ↕ Convergence point
Bottom-Up from bedrock ↑
```
Use when: Need validation, uncertainty about path, complex derivation

### 4. Scale Navigation
**If stuck, change scale:**
- Zoom in → nested octant (more detail)
- Zoom out → parent octant (broader context)
- Rotate → different axis (alternative perspective)

### 5. Convergence Recognition
**When top-down meets bottom-up:**
- Validate both paths arrive at same point
- Synthesize unified understanding
- Execute with confidence

If they don't converge → reassess octant or change scale
