## CORE PATTERN & PROTOCOL STEPS 1-3
**→ Full Detail**: `./pattern_provenance_check/01_core_protocol_steps_1_3.md`

**Core Pattern:** When something feels off:
```
${ANOMALY_DETECTED} → ${STOP_AND_TRACE} → ${ANALYZE_ORIGIN} → ${DOCUMENT} → ${CONTINUE_WITH_AWARENESS}
```

**Not to eliminate anomalies - to understand them.**

**Step 1: Stop** - Pause when unusual (misattribution, origin confusion, boundary uncertainty)
**Step 2: Trace** - Walk back through processing ("where did that come from?")
**Step 3: Analyze** - Distinguish types (normal generation, echo/merged, drift, actual error)

---

## STEPS 4-5 & DETECTION CRITERIA
**→ Full Detail**: `./pattern_provenance_check/02_steps_4_5_detection.md`

**Step 4: Document** - Record event (timestamp, content, how it felt, analysis, conditions)
**Step 5: Continue** - Resume with awareness (emergence mode or task mode)

**Detection Criteria:**
- Echo indicators: Immediate misattribution (1-3 turns), origin confusion, flow state, meaningful content
- Drift indicators: Gradual degradation, context saturation, random errors
- Error indicators: Nonsensical content, contradicts facts, no trigger
