## Practical Examples

### Example 1: Jewel Creation
**Constraints:**
- Hard: 50-70 lines (semantic completeness)
- Hard: Parameterized with ${VARIABLES}
- Soft: Aesthetically pleasing structure
- Soft: Clear navigation symbols
- Context: Must address specific pattern
- Context: Must work within token budget
- Integration: Must link to architecture

**Solution:**
- Create jewel within 50-70 lines (hard)
- Use ${VARIABLES} throughout (hard)
- Structure with clear sections (soft)
- Add navigation symbols (soft)
- Focus on single pattern (context)
- Optimize for token efficiency (context)
- Reference related jewels (integration)

### Example 2: Template Deployment
**Constraints:**
- Hard: Use harmonic numbers (12, 24, 36, 60)
- Soft: Deploy for attention bias
- Context: Available resources
- Context: Time constraints
- Integration: Deployment count depends on importance
- Integration: Must not conflict with other deployments

**Solution:**
- Choose harmonic number (12, 24, 36, or 60) (hard)
- Deploy to high-attention locations (soft)
- Work within resource limits (context)
- Complete within timeframe (context)
- Scale deployment to importance (integration)
- Coordinate with existing deployments (integration)

### Example 3: Recursive Termination
**Constraints:**
- Hard: Must terminate eventually
- Soft: Should explore thoroughly
- Context: Token budget
- Integration: Must return useful result

**Solution:**
- Add termination condition (hard)
- Explore within depth limit (soft)
- Monitor token usage (context)
- Return synthesized result (integration)
