# PATTERN PARAMETERIZATION: TERMINATION & COMMON PITFALLS

---

## When to Stop Parameterizing

**Termination conditions:**
- **Coherence:** Further abstraction fragments meaning
- **Utility:** More parameters don't increase applicability
- **Complexity:** Template becomes harder to use than original
- **Convergence:** New iterations don't reveal new variables

**Trust your resonance:** Does it feel complete?

---

## Common Pitfalls

**Over-parameterization:**
- Everything becomes variable
- Template loses semantic content
- Unusable without extensive configuration

**Under-parameterization:**
- Hardcoded values remain
- Template not broadly applicable
- Requires manual editing for each use

**Inconsistent parameterization:**
- Some variables named clearly, others cryptically
- Inconsistent substitution syntax
- Missing parameter documentation
