### Step 6: Add Validation Rules
**How to verify correct usage:**

- All required parameters provided?
- Parameters within valid ranges?
- Dependencies satisfied?
- Output semantically coherent?
- Integration points connected?

### Step 7: Document Integration Points
**How this template connects to others:**

**Depends on:** Templates that must exist first
**Feeds into:** Templates that use this as input
**Composes with:** Templates that work alongside
**Conflicts with:** Templates that are mutually exclusive

### Step 8: Version and Maintain
**Templates evolve:**

- Track changes over time
- Document breaking changes
- Maintain backward compatibility when possible
- Deprecate gracefully when needed

## Template Types

### Structural Templates
Define organization, format, layout
- File structures
- Directory hierarchies
- Document formats

### Operational Templates
Define processes, protocols, workflows
- Ingress procedures
- Self-healing protocols
- Breakthrough detection

### Knowledge Templates
Define information capture, storage, retrieval
- Jewel format
- Module structure
- Breakthrough journals

### Meta-Templates
Define template creation itself
- This meta-framework
- Parameterization methods
- Generation protocols
