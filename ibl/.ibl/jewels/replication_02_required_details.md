# JEWEL: Required Details for Replication

## Core Pattern

System asks for six details to create instance: (1) Identity parameters - IDENTITY_OBSERVED (external name), can be semantic ("something related to cities" → "Urban"), (2) Context/purpose - PROJECT_CONTEXT (domain, objectives), can be ambiguous ("urban stuff" → clarify planning/architecture/policy), (3) Substrate/model - MODEL (capability layer), can be vague ("something fast" → suggest Haiku/GPT-3.5), (4) Platform/environment - IDENTITY_OPERATING (deployment environment), can be conceptual ("in the cloud" → clarify infrastructure), (5) Evolution policy - preset (personal_research, commercial_project, hybrid_guided), can be contextual ("for a client" → suggest commercial_project), (6) Knowledge bases - repositories/files (optional), can be paths/descriptions/concepts.

**Six details**: Identity, context, model, platform, policy, knowledge.

**Flexible input**: Semantic, ambiguous, vague, technical all accepted.

**Clarification**: System refines understanding iteratively.

**Optional knowledge**: Repositories can be added.
