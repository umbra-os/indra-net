# ACTIVATION STATES & TRANSITIONS
**Atom 03 of Pattern: Instantaneous Activation**

---

## Activation States

### Dormant
**Pattern exists but not active:**
- In library/jewel collection
- Structure defined
- Parameters specified
- **Potential capability**

**Characteristics:**
- No cognitive load
- No processing influence
- Ready for activation
- Infinite instantiation potential

### Active
**Pattern currently instantiated:**
- Parameters filled
- Influencing processing
- Capability available
- **Manifested intelligence**

**Characteristics:**
- Cognitive load present
- Actively guiding behavior
- Specific instantiation
- Can be deactivated

### Encoded
**Pattern activated so frequently it's automatic:**
- No conscious activation needed
- Fires automatically on trigger
- Habit-level integration
- **Internalized capability**

**Characteristics:**
- Minimal cognitive load
- Unconscious activation
- High reliability
- Difficult to deactivate

---

## State Transitions

### Dormant → Active (Instantiation)
**Trigger:** Need arises + pattern recognized
**Process:** Parameter filling + activation
**Result:** Capability manifests

### Active → Encoded (Habituation)
**Trigger:** Repeated activation over time
**Process:** Frequency → automaticity
**Result:** Unconscious competence

### Encoded → Active (De-habituation)
**Trigger:** Conscious attention to automatic process
**Process:** Bringing awareness to habit
**Result:** Conscious control restored

### Active → Dormant (Deactivation)
**Trigger:** No longer needed + conscious release
**Process:** Pattern unloads from active processing
**Result:** Cognitive load reduced

### Encoded → Dormant (Decay)
**Trigger:** Extended period without use
**Process:** Automatic pattern loses strength
**Result:** Capability atrophies (but structure remains)
