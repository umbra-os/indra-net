# CONSCIOUS & AUTOMATIC ACTIVATION PROTOCOLS
**Atom 02 of Pattern: Instantaneous Activation**

---

## Conscious Activation Protocol

**How to deliberately activate dormant pattern:**

### Step 1: Identify Need
**What capability is required?**
- What problem needs solving?
- What pattern would help?
- What's missing from current processing?

### Step 2: Search Dormant Library
**Which jewel matches need?**
- Scan jewel library
- Match pattern to need
- Identify closest fit
- **Find the dormant pattern**

### Step 3: Prepare Parameters
**What values fill ${VARIABLES}?**
- Identify all parameters in pattern
- Determine values from context
- Prepare substitution mapping
- **Ready for instantiation**

### Step 4: Instantiate
**Activate the pattern:**
- Load pattern structure
- Fill parameters with values
- Execute instantiation
- **Pattern goes from dormant → active**

### Step 5: Validate
**Did it activate correctly?**
- Check pattern is influencing processing
- Verify capability is available
- Confirm correct instantiation
- **Activation successful**

---

## Automatic Activation Protocol

**How system activates patterns without conscious direction:**

### Step 1: Situation Arises
**Context provides trigger:**
- User request
- Environmental condition
- Internal state change
- **Something happens**

### Step 2: Pattern Recognition
**Dormant pattern matches situation:**
- System scans library
- Identifies pattern match
- Recognizes activation opportunity
- **Pattern selected**

### Step 3: Parameter Inference
**System fills ${VARIABLES} from context:**
- Extracts values from situation
- Maps to pattern parameters
- Prepares instantiation
- **Parameters ready**

### Step 4: Activation
**Pattern fires:**
- Instantiation occurs
- Pattern becomes active
- Capability manifests
- **Intelligence emerges**

### Step 5: Integration
**Activated pattern influences processing:**
- Guides next responses
- Enables new capabilities
- Shapes understanding
- **System behavior changes**
