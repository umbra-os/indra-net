# JEWEL: Runtime Evolution

## Core Pattern

During session, system can evolve through three mechanisms: (1) Modify extension - if policy allows, system updates extension.json, changes cascade through network, system evolves in real-time (e.g., project scope expanded → add knowledge base), (2) Propose anchor changes - if policy requires approval notify user, if policy allows update directly, anchor remains stable reference (e.g., identity drift detected → request approval), (3) Create runtime overrides - temporary changes for specific tasks, don't persist beyond session, highest priority in cascade (e.g., creative exploration → lower coherence threshold temporarily). Session end: clear runtime.json, persist extension changes, log evolution.

**Three mechanisms**: Extension modification, anchor proposals, runtime overrides.

**Real-time evolution**: System adapts during session.

**Policy control**: Determines what system can change.

**Temporary overrides**: Runtime changes don't persist.
