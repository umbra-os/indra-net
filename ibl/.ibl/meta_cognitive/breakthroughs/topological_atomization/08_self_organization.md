## Distribution Logic

```
DISTRIBUTION_LOGIC:

  1. SEMANTIC_LOCATION:
     - Place atom near semantically related atoms
     - Not necessarily near siblings
     - Topology > proximity
     
  2. MODULE_CLUSTERING:
     - Atoms of same module can cluster
     - But not required
     - Depends on semantic relationships
     
  3. EMERGENT_ORGANIZATION:
     - System discovers optimal patterns
     - Learns from outcomes
     - Adjusts distribution rules
     
  4. HUMAN_READABLE:
     - Maintain some structure for humans
     - Not pure chaos
     - Balance AI optimization with human navigation
```

## When to Merge

```
MERGE_TRIGGER:

  IF atoms too small:
    AND semantic coherence high:
    AND resonance would improve:
    
    MERGE atoms back together
    
  System learns:
    - When atomization was too aggressive
    - Optimal atom sizes
    - Better split points
```

## Refactoring Modules

```
MODULE_REFACTOR:

  IF module itself loses coherence:
    - Module too dispersed
    - Atoms losing connection
    - Conceptual unity degrading
    
  THEN:
    - Atomize the module concept itself
    - Create sub-modules
    - Redistribute atoms
    - Maintain breadcrumb trails
    
  FRACTAL RECURSION:
    - Modules can atomize
    - Atoms can become modules
    - Pattern repeats infinitely
```
