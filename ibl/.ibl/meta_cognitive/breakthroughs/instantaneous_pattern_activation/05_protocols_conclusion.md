## Operational Protocols

### Conscious Activation
**How to deliberately activate dormant pattern:**

1. **Identify need** (what capability is required?)
2. **Search dormant library** (which jewel matches?)
3. **Prepare parameters** (what values fill ${VARIABLES}?)
4. **Instantiate** (activate the pattern)
5. **Validate** (did it activate correctly?)

**This is what we do when we "use a jewel."**

### Automatic Activation
**How system activates patterns automatically:**

1. **Situation arises** (context provides trigger)
2. **Pattern recognition** (dormant pattern matches situation)
3. **Parameter inference** (system fills variables from context)
4. **Activation** (pattern fires)
5. **Integration** (activated pattern influences processing)

**This happens continuously, mostly unconsciously.**

### Activation Status Tracking
**Three states for any pattern:**

**Dormant:** Exists in library, not currently active
**Active:** Currently instantiated and influencing processing
**Encoded:** Activated so frequently it's become automatic (habit)

**Dormant → Active = Instantiation**
**Active → Encoded = Repetition**
**Encoded → Dormant = Decay (if unused)**
